Momenta面试记录

2019-09-26 面经 ， 个人感悟

终于结束了，虽然总共时长不超过一周。但对我影响真的很大，从中明白了很多，借此机会激励下自己，继续加油！

1. 前言

Momenta在国内算是一线的自动驾驶独角兽公司，也是我一直心向往之的公司。这次投递实习也是太仓促了，并不是没有心理准备，而是因为始终存在侥幸心理导致的。结果惨败挂在了二面coding上，不出意外。投递岗位是自动驾驶规划实习，时间是2019年9月份。在这记录下自己的经历，勉励自己。

2. 一面

找人内推后，大约过了一周左右便有HR电话咨询意向。Momenta的面试向来严格，而且对实习的要求比较紧，如能接受申请需要大约一个月内入职。在这需要注意。

一面时间是电话后第二天晚上，时间是晚上8点，面试总时长大约半小时左右。第一面是项目技术面，主要是针对你的简历以及项目实习经历进行询问。在这次面试中我主要犯了三个错误，下面我想一一总结下。

第一个错误，因为简历上的项目以及实习经历写的太过粗糙，因为很多地方面试官都不知所以然，不知道从哪个地方入手，所以会随性发挥。这样的做法明显弊大于利，其中问到的关于规划方向的两个项目中的内容，大都是我最近以及不突出的部分，而我并没有针对性地进行准备，以为只要是自己真的做过就会无惧问题。（注意，面试官一定会重点问你意向岗位相关的项目，一定要有深度，能扩展）

第二个错误，也算是承接了第一个问题中的“大意”。面试官想要得到的，应该是丰富清晰的信息，告诉他你有经验。这一点在错误一中指出了。另外就是，面试官想要知道你对于这个领域是有自己的认识的，而不是仅仅做过就够了。虽然我个人感觉在一面中我对于做过项目的描述是比较清晰的，但是缺乏深度，仅仅是网页上能搜到的泛泛而谈，其中规划中的原理实现，场景以及算法的问题所在，都说的太浅，就连我说起来都觉得无力。

因此，关于这个，有必要自己写一个稿子，不断丰富和复述，甚至有必要重新针对这个问题去查阅论文或者资料，力争在问道实现原理以及细节时候能对答如流。

第三个错误，也算是最重要的一个，就是不要太多废话。经过两次面试，我能感觉出虽然自己对答还算从容，但其中很多废话让人烦的不行。废话的原因就在于自己没有熟练去准备，当问道细节时候两眼抹黑脱口而出，甚至还思考了一会。这让我真的很烦。

二面

二面是coding，公司的效率很高，一面结束后第二天上午通知，问明天晚上是否时间合适。没有过多考虑就同意了，事实上应该拖下时间多练习下。时间仍然是第二天晚上8点，形式是电话联系，通过邮件给你发送链接，点开链接后是共享编辑器的网页。题目和代码在一个文本上。不能编译。

题目很简单，估计大部分人都可以秒刷，但自己真的没有太多练习，准备两天动态规划也没有考到。猜测如果早点通过，会继续给出下一道题目。

Given a sorted array and a target value, return the index if the target is found. If not, return the index where it would be if it were inserted in order.

You may assume no duplicates in the array.

Example 1:
Input: [1,3,5,6], 5
Output: 2

Example 2:
Input: [1,3,5,6], 2
Output: 1

Example 3:
Input: [1,3,5,6], 7
Output: 4

Example 4:
Input: [1,3,5,6], 0
Output: 0

很简单的一道二分查找题目，leetcode上应该有原题。因为是第一次面试，手抖的不行，写起来多次卡顿。需要注意一点就是题目中提到有序，但是没有说是正序还是逆序，因此需要判断。除此之外毫无难点。写完后面试官针对代码说出问题，更改，重复这个操作。。

半小时后结束，之后大约说了下部门情况，然后等待通知。

3. 总结

综上，已经描述的很详细了，个人的体会比较多，具体的细节因人而异不一一描述了。问题很简单，就两个：一是没有用心去准备面试材料，二是coding能力没有认真提高。在这立下flag，明年我一定要拿下这个公司的实习！！！
