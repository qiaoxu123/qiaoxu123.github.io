Ubuntu环境下OOQP优化库的基本使用

发布于 2020-01-06 | 分类于 最优化 、 工程实现 | 11分钟 | 2346字数

OOQP是一个强大的开源凸优化库，支持C++, Maltab调用。学习过程中发现网上的参考资料寥寥无几，在这根据官方的指导手册和Example简单介绍说明下。

1. OOQP公式中相关定义

OOQP认可的“通用”二次规划公式如下：

min⁡12xTQx+cTx subject to Ax=b,d≤Cx≤f,l≤x≤u\begin{aligned} &\min \frac{1}{2} x^{T} Q x+c^{T} x \\ \text { subject to } &A x=b, \quad d \leq C x \leq f, \quad l \leq x \leq u \end{aligned}  subject to ​min21​xTQx+cTxAx=b,d≤Cx≤f,l≤x≤u​

其中 QQQ 是 n×nn×nn×n 的正半定矩阵， AAA 是 ma×nm_a×nma​×n 矩阵，CCC 是 mc×nm_c×nmc​×n 矩阵，所有其他量都是适当大小的向量。 l,u,dl,u,dl,u,d 和 fff 的某些元素的大小可能是无限的。 也就是说，CxCxCx 和 xxx 的某些分量可能没有上限和下限。

在这，Ax=bA x=bAx=b 是线性等式约束，d≤Cx≤f,l≤x≤u\quad d \leq C x \leq f, \quad l \leq x \leq ud≤Cx≤f,l≤x≤u 为不等式约束。

稀疏矩阵由三个数据结构表示 -- 两个整数向量和一个双精度向量，它们的长度都相同。对于(通用)矩阵AAA ，这些数据结构为irowA，jcolA和dA。稀疏矩阵 AAA 中的非零元素综述总数为nnzA。AAA 的第k个非零元素出现在第irowA[k]行和第jcolA[k]列，其值为dA[k]。务必注意：行和列从零开始编号。 对于对称矩阵 QQQ，仅在irowQ，jcolQ和dQ中指定矩阵下三角的元素。

下面具体介绍下程序中用到的几个元素：

  • c : 目标函数中的线性项，长度为nx的向量
  • nx : 变量的数量，即向量x的长度。同时也是输入向量c，xlow，ixlow，xupp，ixupp，x，gamma 和 phi 的长度。
  • irowQ，jcolQ，dQ 包含目标函数二次项的nnzQ下三角元素
  • xlow，ixlow 是 x 的下限。 对应公式中的下界向量 lll 。如果 xxx 的元素 kkk 上有界，则 xlow [k] 应设置为 lkl_klk​ 的值，ixlow [k] 设置为1。 否则，两个数组的元素 k 都设置为零
  • xupp，ixupp是 xxx 的上限，对应公式中的上界向量 uuu 。定义方式与 xlow, ixlow 相同
  • irowA，jcolA，dA 综合描述了线性等式约束矩阵 AAA 的 nnzA 个非零元素
  • irowC，jcolC，dC 综合描述了线性不等式约束的矩阵 CCC 的nnzC个非零元素
  • b : 公式中线性等式约束的右侧向量 bbb
  • nnzQ：矩阵下三角元素的个数，同时对应着irowQ，jcolQ的长度
  • my : 线性等式约束的个数，也是向量 AAA 与 bbb 的长度
  • mz : 不等式约束的个数，同时对应着clow，iclow，cupp, icupp的长度
  • nnzC ： 多项式不等式约束的个数
  • clow, iclow ：非等式约束的下限
  • cupp, icupp ：非等式约束的上限

2. C++ 对OOQP库函数的调用

首先，程序中需要包含如下头文件

#include "QpGenData.h"
#include "QpGenVars.h"
#include "QpGenResiduals.h"
#include "GondzioSolver.h"
#include "QpGenSparseMa27.h"

使用C++调用OOQP时，用户必须创建多个对象并依次调用多个方法。相比C语言的函数调用更加复杂，但也更加灵活。通过改变创建的对象的类别，可以为各种类型的QP生成定制的求解器。在这我们主要介绍公式中的默认求解器。这种情况的完整调用顺序如下所示。

QpGenSparseMa27 * qp
        = new QpGenSparseMa27( nx, my, mz, nnzQ, nnzA, nnzC );
QpGenData * prob
        = (QpGenData * ) qp->makeData( /* parameters here */);
QpGenVars * vars
        = (QpGenVars *) qp->makeVariables( prob );
QpGenResiduals * resid
        = (QpGenResiduals *) qp->makeResiduals( prob );
GondzioSolver * s = new GondzioSolver( qp, prob );

s->monitorSelf();
int status = s->solve(prob,vars, resid);

第一行初始化类QpGenSparseMa27的新求解公式qp，它是ProblemFormulation的子类。此类的定义确定了如何存储问题数据，问题变量将如何存储和操作，以及如何求解线性系统。 我们的子类QpGenSparseMa27用来对公式进行实现，其中定义问题的稀疏矩阵存储在稀疏（非密集）矩阵中，并且定义内点方法步骤的大型线性系统将通过使用Harwell Subroutine Library中的MA27包进行求解。

在第二行，使用创建的对象qp中的makeData函数创建包含问题数据的向量和矩阵。实际上，qp包含makeData方法的不同版本，这可以通过其不同的参数列表来区分。详见官方指导手册，在这不具体介绍。

QpGenData * prob
        = (QpGenData * ) qp->makeData(  c, krowQ, jcolQ, dQ,
                                        xlow, ixlow, xupp, ixupp,
                                        krowA, jcolA, dA, bA,
                                        krowC, jcolC, dC,
                                        clow, iclow, cupp, icupp);

第四行和第五行分别调用makeVariables和makeResiduals创建了一些对象，这些对象存储了问题变量和残差，这些问题变量和残差衡量了给定点相对于各种最优性条件的可行性。对象vars包含公式的原始变量（包括x）和对偶变量（拉格朗日乘数）。这些变量的名称分别为vars-> x，vars-> y等。当找到问题的近似解时，此对象中的所有数据元素都将具有较小的值，这表明所讨论的点近似满足所有最优性条件。

下一步是创建用于实际求解QP的求解器对象。这通过以下调用来执行。

GondzioSolver * s = new GondzioSolver( qp, prob );

在我们的示例中，我们然后调用方法s-> monitorSelf()来告知求解器它应在求解器运行时将摘要信息打印到屏幕上。(如果省略此行，则求解器将不输出打印信息。)最后，我们通过调用s-> solve(prob，vars，resid)调用算法来解决问题。如果求解器能够计算近似解，则该例程的返回值将为零，该近似解可在对象vars中找到。如果无法解决问题，则求解器将返回非零值。负值表示遇到了一些错误，例如内存不足错误。有关OOQP终止标准以及可能以ierr返回的正值的描述，请查看官方的指导手册。

3. C++代码实例

下面以官方的一个Demo举例，来进一步说明OOQP的使用过程：

mimimize⁡4+1.5x1−2x2+12(8x12+4x1x2+10x22) subject to 2≤2x1+x2≤∞−∞≤−x1+2x2≤60≤x1≤200≤x2≤∞\begin{array}{cl} {\operatorname{mimimize}} & {4+1.5 x_{1}-2 x_{2}+\frac{1}{2}\left(8 x_{1}^{2}+4 x_{1} x_{2}+10 x_{2}^{2}\right)} \\ {\text { subject to }} & {2 \quad \leq 2 x_{1}+x_{2} \quad \leq \infty} \\ & {-\infty} {\leq-x_{1}+2 x_{2}\leq 6} \\ & {0 \quad \leq}{x_{1} \quad \quad \quad \quad \leq 20} \\ & {0 \quad \leq} {x_{2} \quad \quad \quad \quad \leq \infty} \end{array} mimimize subject to ​4+1.5x1​−2x2​+21​(8x12​+4x1​x2​+10x22​)2≤2x1​+x2​≤∞−∞≤−x1​+2x2​≤60≤x1​≤200≤x2​≤∞​

与第一部分中提到的OOQP“通用”二次规划公式相对应，对于该问题的最小化求解，其实就是最小化该函数：

f=4+1.5x1−2x2+12(8x12+4x1x2+10x22)f = {4+1.5 x_{1}-2 x_{2}+\frac{1}{2}\left(8 x_{1}^{2}+4 x_{1} x_{2}+10 x_{2}^{2}\right)} f=4+1.5x1​−2x2​+21​(8x12​+4x1​x2​+10x22​)

其中 12xTQx\frac{1}{2} x^{T} Q x21​xTQx 即 12(8x12+4x1x2+10x22)\frac{1}{2}\left(8 x_{1}^{2}+4 x_{1} x_{2}+10 x_{2}^{2}\right)21​(8x12​+4x1​x2​+10x22​) ，cTxc^{T} xcTx 即 4+1.5x1−2x24+1.5 x_{1}-2 x_{2}4+1.5x1​−2x2​。该函数中不存在等式约束，因此 Ax=bA x=bAx=b 中参数均为0。需要注意的是，不等式约束按照OOQP的标准划分为了两部分，一部分是各个变量的不等式约束，另一部分是多项式的不等式约束。

即 d≤Cx≤fd \leq C x \leq fd≤Cx≤f 对应 2≤2x1+x2≤∞{2 \leq 2 x_{1}+x_{2} \leq \infty}2≤2x1​+x2​≤∞ 和 −∞≤−x1+2x2≤6{-\infty} {\leq-x_{1}+2 x_{2}\leq 6}−∞≤−x1​+2x2​≤6。

l≤x≤ul \leq x \leq ul≤x≤u 对应 0≤x1≤20{0 \leq}{x_{1} \leq 20}0≤x1​≤20 和 0≤x2≤∞0 \leq x_{2} \leq \infty0≤x2​≤∞

将目标函数也相应的约束转化为代码，即：

const int nx = 2;
double c[] = {1.5, -2};   // 目标函数中的线性项，长度为nx的向量

double xupp[] = {20, 0};  
char ixupp[] = {1, 0};

double xlow[] = {0, 0};
char ixlow[] = {1, 1};

const int nnzQ = 3;       // 数目与矩阵的下三角矩阵的个数相对应
int irowQ[] = {0, 1, 1};  // 对称矩阵Q，仅在irowQ,jcolQ和dQ中指定
int jcolQ[] = {0, 0, 1};  // 矩阵的下三角元素
double dQ[] = {8, 2, 10};

int my = 0;               // 线性等式约束的个数
double *b = 0;            // 线性等式约束的右侧向量b
int nnzA = 0;             // nonze-ros总数
int *irowA = 0;           // 等式约束中的A
int *jcolA = 0;
double *dA = 0;

const int mz = 2;         // 不等式约束的个数
double clow[] = {2, 0};
char iclow[] = {1, 0};

double cupp[] = {0, 6};
char icupp[] = {0, 1};

const int nnzC = 4;       // 多项式不等式约束的个数
int irowC[] = {0, 0, 1, 1};
int jcolC[] = {0, 1, 0, 1};
double dC[] = {2, 1, -1, 2};

最后按照第二部分的函数调用进行实现即可：

  QpGenSparseMa27 *qp = new QpGenSparseMa27(nx, my, mz, nnzQ, nnzA, nnzC);

  QpGenData *prob = (QpGenData *)qp->copyDataFromSparseTriple(
      c, irowQ, nnzQ, jcolQ, dQ, xlow, ixlow, xupp, ixupp, irowA, nnzA, jcolA,
      dA, b, irowC, nnzC, jcolC, dC, clow, iclow, cupp, icupp);

  QpGenVars *vars = (QpGenVars *)qp->makeVariables(prob);
  QpGenResiduals *resid = (QpGenResiduals *)qp->makeResiduals(prob);

  GondzioSolver *s = new GondzioSolver(qp, prob);

  if (!quiet)
    s->monitorSelf();
  int ierr = s->solve(prob, vars, resid);

  if (ierr == 0) {
    cout.precision(4);
    cout << "Solution: \n";
    vars->x->writefToStream(cout, "x[%{index}] = %{value}");
  } else {
    cout << "Could not solve the problem.\n";
  }
